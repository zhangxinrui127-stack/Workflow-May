6.2 关键设计决策

决策 选择 理由
信息源策略 NotebookLM (一手) + Web (补充) NotebookLM 提供 SEC Filing 级别精确数据；Web 提供实时市场数据
并行粒度 每 turn 3-4 个工具并行 最大化吞吐量，减少 turn 数
模块间依赖 允许跨模块并行（M4+M5 同时；M6+M7 同时） 模块间有主题重叠但无严格依赖
产出格式 Markdown（非 PPT） 中间产物更易审阅和修改；PPT 为最终产出
三表模型处理 不在 Markdown 中展开完整 model 用户明确指示放 Excel；报告聚焦假设和结果
任务管理 TaskCreate/TaskUpdate 跟踪 对 8 模块进度提供可见性

6.3 NotebookLM Prompt 设计模式

每个 NotebookLM 查询遵循以下模式：

notebooklm ask "[Context Frame]: Provide detailed [topic].
Specifically address:
1) [Specific question with quantitative anchor]
2) [Specific question with comparison frame]
3) [Specific question with time series request]
...
