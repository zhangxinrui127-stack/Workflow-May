摘要

本次协作围绕一份包含 8 个模块的 CoreWeave (CRWV) 投资研究报告展开了四轮深度审校与迭代。整个过程从原始报告的数字校对出发，逐步深入至逻辑审查、事实验证、关键假设挖掘，最终完成了一份经过全面修正和增强的终稿。

第一轮（数字校对 + 逻辑审查）：对 8 个模块中 300+ 个数据点进行系统性核实，发现 3 个严重错误（EV 计算双重扣减、FY2023 收入数据混淆、W&B 收购价格错误）、7 个中度问题和 3 个轻微问题。同时为每个模块标注逻辑审查意见，共产出 49 个审查点。通过 4 个并行研究代理（财务、合同、行业、治理）进行独立验证，产出 8 份 Reviewed Module 文件和 1 份数字校对报告。

第二轮（文件整理）：将带有 "Reviewed" 标识的 8 个 markdown 文件整理至 reviewed-report/ 子文件夹。

第三轮（关键问题深度验证）：用户要求对 49 个逻辑审查点逐一进行二次事实收集与结论判定，并特别深挖三个专题——(1) 债务融资比例假设的实证推导，(2) 利息支出与美联储降息路径的关系，(3) FY2026+ 客户级收入预测。再次启动 4 个并行研究代理（逻辑审查提取、Fed 利率、客户算力支出、债务融资结构），产出 key-questions-validation.md 和 CRWV_Customer_Compute_DeepDive.md。

第四轮（追问与终稿整合）：用户针对专题一提出两个追问——(a) 股权融资倾斜是否可行？(b) Capex 放缓对估值的实际影响是什么？Claude 完成深度分析后，将全部验证结论（49 个审查点 + 2 个追问分析）以连贯叙事方式 incorporate 进 Final Report，核心修正包括：Base Case 目标价从 $179 下调至 $155，期望值从 $168 下调至 $139（+17%），Distress 概率从 5% 上调至 10%。修正后的 12 个月折现期望值 ~$121 与华尔街共识 $120-125 高度吻合。

用户追问（2 个关键问题）

Claude 分析

问题 1：股权融资倾斜的可行性

分析了三条可行路径：

  1. 可转债（最优）：1.75% 几乎免费资金，再发 $3-5B 可行
  2. 战略投资者：NVIDIA/Jane Street 模式，$2-3B 可期，信号效应最小
  3. 二次增发：$7.5-9B 对应 12-14% 稀释，技术可行但信号负面

核心结论：向股权倾斜不是管理层主动选择，而是 covenant 约束下的被迫行为。累计潜在稀释（含现有可转债）达 18-25%。

问题 2：Capex 放缓悖论

构建了分析框架：

  • "好的放缓"（backlog 继续增长）→ 类比 Amazon 2015-2017，股价 3.3x
  • "坏的放缓"（backlog 同步下滑）→ 类比 Global Crossing 2000-2001，估值崩塌
  • "被迫放缓"（融资受限）→ 最坏情景，股价可能跌至 ~$26

核心判断：对于 Net Leverage 5.65x、CDS 665bps 的公司，风险折价的消除 > 增长溢价的消退。Capex 放缓对股价的净效应大概率为正面，前提是 backlog 仍在增长。

关键监控指标不是 capex 增速本身，而是 capex 放缓时 backlog 的变化方向。

三、协作方法论总结

工作流架构

原始 8 模块
    │
    ├─ Phase 1: 并行研究代理 (4x) → 数字校对 + 逻辑审查
    │   └─ 产出: 数字校对报告 + 8 份 Reviewed Module + Final Report v1
    │
    ├─ Phase 2: 文件整理 → reviewed-report/ 子文件夹
    │
    ├─ Phase 3: 并行研究代理 (4x) → 49 个审查点二次验证 + 3 专题深挖
    │   └─ 产出: key-questions-validation.md + Customer DeepDive
    │
    └─ Phase 4: 用户追问 (2 个) → 分析 → 全量整合
        └─ 产出: Final Report v2（终稿）

关键方法

  1. 并行代理模式：两轮共部署 8 个独立研究代理，分别负责财务、合同、行业、治理、利率、客户、债务、逻辑提取，大幅提升事实核查的覆盖面和效率
  2. 分层验证：第一轮做数字校对（表层），第二轮做逻辑验证（深层），第三轮做假设挖掘（根层）
  3. 连贯整合原则：用户明确要求最终报告必须"完整通顺连贯"，而非松散的专题拼接——所有验证结论以自然叙事方式融入各模块对应位置
  4. 双向互动：用户在 Phase 4 提出两个高质量追问，驱动了对 capex 放缓悖论和股权融资可行性的深入分析，这是纯 AI 审校无法自发产生的洞察方向

质量控制

  • 49 个审查点中 0 个被标注为 "Needs Revision"（原结论有误），说明第一轮审校质量较高
  • 6 个 "New Finding" 均来自二次验证中的新增信息（CDS 定价、传导链概率、Jane Street 合同确认等）
  • 最终估值修正（期望值 $168→$139）使报告结论与华尔街共识从显著背离变为高度吻合，增强了报告的可信度

四、核心发现与投资结论演变

估值演变轨迹

阶段 Base Case 期望值 vs $118
原始报告 $179 ~$168 +42%
数字校对后（EV 修正） ~$170 ~$160 +35%

关键认知转变

  1. 从乐观到审慎：原始报告对净债务增长估计不足（$25B→$33-35B），导致 Base Case 高估约 15%
  2. 从单维到多维风险：新增风险传导链分析，将流动性风险概率从"低"上调至"低-中"（条件概率 10-15%），CDS 665bps 提供市场验证
  3. 从粗放到精细：利息支出从 "$2.5-3.0B" 精确至 "$2.0-2.8B（中位 $2.4B）"，并附加 SOFR 敏感性表
  4. 从模糊到具体的增长路径：新增客户级收入展望表，明确 Anthropic 为最大上行意外来源、Microsoft 为最大下行确定性风险
  5. 识别核心悖论：Capex 放缓悖论——EBIT 转正需要 capex 放缓但可能被解读为增长见顶。结论：对高杠杆公司，风险折价消除 > 增长溢价消退，净效应大概率正面
