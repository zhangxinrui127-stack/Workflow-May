摘要

本次会话记录了 pptx-research skill 的协作创建过程 -- 一个用于生成高密度、信息丰富的投研/咨询场景 PPTX 报告的 Claude Code skill。主要阶段如下：

  1. Skill 需求定义 (16:08): User 提出了完整的 skill 需求，包括 5 步交互式流程（内容输入 -> 语言格式 -> 大纲 -> Storyline -> 渲染），强调内容强管控、高信息密度、非路演型 deck。
  2. Plan Mode 协作设计 (16:09-16:36): Claude 进入 Plan Mode，探索现有 pptx skill 结构，设计新 skill 架构。用户提出三处修正：(1) outliner 是可选副标题而非要点列表；(2) 逻辑审查应在 visual elements 之前；(3) 需要断点恢复机制。
  3. Skill 编码实现 (16:44-17:06): 计划获批后，Claude 创建了完整的 skill 目录结构，编写了 SKILL.md 主文件和 8 个 reference 文件（step-0 到 step-5、chart-patterns、layout-catalog），共 10 个文件。
  4. 首次测试 -- CRWV Module1 (17:07-17:28): 用户用"帮我做一个咨询风格ppt"触发 skill，选择 CRWV 行业研究报告作为内容源，选择英文输出、麦肯锡配色（蓝黑色调），生成了 8 页模板预览。
  5. 迭代反馈 Round 1 -- 内容密度 (18:03): 用户指出 P5-P8 内容太少，要求每页至少 200-300 词或 100 词+图表；表格样式不好看，边框多余，左右间距不对称。Claude 将这些规则写入 skill 的 G8 和 G9 规则。
  6. Context 恢复 (18:08): Session 因 context 用尽被 continue，Claude 基于 summary 自动恢复工作，重新生成模板。
  7. 迭代反馈 Round 2 -- 内容溢出 (18:13): 用户反馈内容严重溢出，提供了一个 BCG 风格的 template.pptx 作为排版参考。Claude 分析了参考文件的布局坐标，校准了 commentary 密度（heading + 3-4 个简洁 bullet points）。
  8. 迭代反馈 Round 3 -- 风格优化 (18:27-18:41): 用户提出 PPT 制作原则（字体统一、对称 gap、图表居中），并反馈整体太靠上、专业度不够、风格太偏 roadshow。Claude 将字体最小设为 12pt，去掉巨大数字展示，改为分析评估表格风格。
  9. Memory 保存 (18:35): Claude 将 PPTX 密度校准和设计原则保存到 memory 系统，供未来 session 使用。
  10. 指令回顾 (20:54): 用户要求 Claude 拼接所有手动输入的指令返回。

我们来 create 一个 pptx 文件生成的 skills，你可以打开 plan mode 我们一起协作。这个 pptx skills 适用于用户需要对内容需要强管控，生成投研/咨询场景的高密度（informative），多图表、非路演型 deck 的生成场景。这个 skills 加载之后，需要用户输入完整的报告内容（doc/docx/md 文件，最差也需要一个 prompt）需要先提示用户 pptx 制作依据的内容输入选项，文件上传，如果内容过少需要提示内容 out of control 的风险（你可以用 web-search 自己添加）否则应该告知用户，该 pptx 会严格依照用户提供内容来制作。接下来需要依次和用户确认 1）输出 pptx 语言（支持 中文/english），2）整体大纲（根据用户上传内容，大致分为几个章节/每个章节名称/每个章节估计页数）；3）大纲确认后，每一页的 title 和 outliner 和 key message （title 和 outliner 会实际显示在每一页 slide 的上方，但是 key message 是这一页大致想呈现的内容/核心论点/论据等，不一定是最终在 slide 上的原话，还会被分解成 key visual elements）；4）storyline（整个 deck 每一页的 title/outliners）确定之后，每一页都需要和用户依次敲定 key visual elements（slide 的左右/上下，每个 block/segment 分别放什么内容，是图/表/bullet points/示意图，还是其他）；5）整个 deck 的内容确定后，最后一步是格式/format 的确定和渲染，这一步需要和用户确定色系 / 字体 /font size （正文、标题、source 等）；注意在语言确定好之后先生成 5 到 10 页 template 给用户参考，等用户确认之后再将之前确定的内容渲染到 pptx 中，注意每个章节生成完成之后都得到用户的确定再继续渲染下一个章节，最小化返工和 token 浪费
