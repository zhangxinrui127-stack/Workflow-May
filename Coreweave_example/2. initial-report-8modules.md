| **总计** | **~63 次** | — |

### 4.2 NotebookLM vs Web 搜索的分工

| 数据类型 | 首选工具 | 理由 |
|---|---|---|
| 公司财务数据 (revenue, EBITDA, capex) | NotebookLM | 10-K/Earnings Call 已导入，数据精确 |
| 合同条款、RPO、covenant | NotebookLM | SEC Filing 一手数据 |
| 管理层背景、治理结构 | NotebookLM | S-1/Proxy Statement 已导入 |
| 技术架构 (MFU, SUNK, DPU) | NotebookLM | 白皮书已导入 |
| 市场规模、增速预测 | Web 搜索 | 需要最新第三方研究机构数据 |
| 竞对信息 (Lambda, Crusoe, Nebius) | Web 搜索 | NotebookLM 中竞对信息有限 |
| GPU 价格趋势 | Web 搜索 | 需要实时市场数据 |
| AI Lab ARR (Anthropic, OpenAI) | Web 搜索 | 非 CoreWeave SEC 文件范围 |
| Hyperscaler capex 计划 | Web 搜索 | 需要各公司最新财报数据 |
| 分析师目标价、做空报告 | Web 搜索 | 需要最新卖方/买方观点 |

### 4.3 并行执行策略

AI 在整个研究过程中大量使用并行工具调用以提高效率：

- **Round 1 典型模式**：2 个 `notebooklm ask` + 2 个 `WebSearch` 同时发起
- **跨模块并行**：Module 4 + 5 同时研究（3 个工具并行）；Module 6 + 7 同时研究（3 个工具并行）
- **写入并行**：Module 4 和 5 的 `Write` 操作在同一 turn 完成
